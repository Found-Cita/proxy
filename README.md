# proxy

Локальный SOCKS5-прокси на `127.0.0.1:4444` с пробросом через upstream SOCKS5 (`IP:PORT`).

## Что делает

- Поднимает SOCKS5 на `127.0.0.1:4444`
- Для каждого подключения ходит через указанный upstream SOCKS5
- Поддерживает `SOCKS5 CONNECT` + `NO AUTH`
- Повторный запуск `./proxy NEW_IP:PORT` не поднимает новый процесс, а обновляет upstream в уже запущенном процессе
- При смене upstream все активные туннели закрываются, чтобы новые подключения сразу шли через новый адрес

## Как пользоваться

Первый запуск:

```bash
./proxy 203.0.113.10:1080
```

С авторизацией upstream:

```bash
./proxy 203.0.113.10:1080 user:password
```

Прокси начнет слушать:

- `127.0.0.1:4444`

## Смена upstream без перезапуска процесса

Просто вызови бинарник еще раз с новым адресом:

```bash
./proxy 204.0.115.10:1080
```

Или со сменой на новый логин/пароль:

```bash
./proxy 204.0.115.10:1080 newuser:newpassword
```

Что произойдет:

- running-процесс получит команду обновления upstream
- старые активные соединения будут закрыты
- новые соединения пойдут через новый upstream

## Остановка

Остановить процесс:

```bash
pkill -f '^./proxy '
```
